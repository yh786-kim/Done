## 2. 누가 쓰나요 (사용자)
주로 어떤 사람이 사용하나요? 여러 종류면 다 적어 주세요.
> 예: 물건을 나누는 사람 / 물건을 받는 사람 / 관리자

- 답: 요청하는 사람도 쓰고 요청 받는 사람도 쓰고

## 3. 왜 필요한가요 (문제 / 목적)
지금 어떤 불편함이 있고, 이 앱으로 무엇을 해결하고 싶은가요?

- 답: 예를들어 엄마가 매일 아침마다 양치 했니? 세수는 했고? 가방은 챙겼니? 이런걸 말로 하지 않고 앱을 통해 특정 시간에 물어보고 답을 확인하고 싶어

## 4. 핵심 기능 (꼭 있어야 하는 것)
가장 중요한 기능 3~5개만. 있으면 좋은 것 말고 **없으면 안 되는 것** 위주로.
> 예: 회원가입/로그인, 물건 등록(사진+설명), 목록 보기, 채팅

- [ 회원가입 절차가 아주 심플했으면 좋겠어. 닉네임/전화번호 이렇게만 등록했으면 좋겠고. 전화번호 인증을 간단하게 할 수 있을까? 복잡하기 않게. 예를들어 등록한 전화면호로 링크 문자를 보내주고 클릭하면 승인 된다던지..  ]
- [ 사용자는 간단하게 2개 역할을 가질 수 있어. 요청 하는사람과 요청 받는 사람으로. 요청 하는사람의 입장에서는 상대방의 전화번호를 기준으로 수신자를 지정하고 일시/확인받을 내용을 등록하는거지. 그리고 내가 요청한 사람의 목록이 있고 각각에 대해 그 사람의 시간대별 확인받는 내용과 했고 안했고에 대한 결과를 알 수 있었으면 좋겠어. 요청 받는 사람으로서의 입장에서는 누군가 내게 요청한 내용을 시간대별로 확인할 수 있고 그 시간이 되면 알림이 왔으면 좋겠고 거기에 대해 했다 라는 피드백을 줬으면 좋겠어. ]
- [ 내가 물어보는 확인 내용을 켜고 끄고 할 수 있음 좋겠어. ]

## 6. 화면 흐름 (대략)
사용자가 앱을 켜고 목표를 이루기까지의 순서를 간단히.
> 예: 앱 실행 → 로그인 → 물건 목록 → 상세 보기 → 채팅하기

- 답: 앱 실행 -> 로그인 -> 계정이 없는 경우 별명+전화번호로 가입 -> 2개 중 선택, 1번: 내가 할일??, 2번: 니가 할일!! -> 1번을 선택하면 시간 순서대로 나에게 확인을 요청한 목록을 확인할 수 있고 했다 안했다를 표시할 수 있어, 이건 날짜별로 페이지 확인을 해야겠지 -> 2번을 선택하면 시킬 사람을 등록할 수 있어(전화번호 또는 닉네임으로 검색 등록) 신규로 대상자를 등록하고 요청 확인 내용을 등록할 수 있어, 이미 등록된 사람은 이미등록한 내용을 확인할 수 있어, 했는지 안했는지 확인할 수 있고 아직 도래하지 않은 내용은 켜고 끌 수 있어.

> 예: "당근마켓처럼 심플하고 밝게", 메인 색상은 초록색

- 답: 당근마켓처럼 심플하고 밝게, 메인 색상은 당근마켓처럼 초록/주황?
